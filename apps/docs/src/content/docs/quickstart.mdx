This walks you through getting OMA running — choose the path that fits your environment.

## Pick Your Path

<Tabs>
<TabItem label="🐳 Docker (recommended)">

Fastest way from zero to a running agent. Requires only Docker.

### Prerequisites

| Tool | Version | Why |
|---|---|---|
| [Docker](https://docs.docker.com/get-docker/) | Latest | Runs the sandbox, database, and vault sidecar |
| [curl](https://curl.se) | Any | Test the API from the terminal |

### Steps

<Steps>

1. **Clone the repository.**

   ```bash
   git clone https://github.com/duyet/oma.git
   cd oma
   ```

2. **Configure environment variables.**

   ```bash
   cp .env.example .env
   ```

   Generate two required secrets:

   ```bash
   export BETTER_AUTH_SECRET=$(openssl rand -hex 32)
   export PLATFORM_ROOT_SECRET=$(openssl rand -base64 32)
   ```

   Add them to `.env`:

   ```bash
   # .env
   BETTER_AUTH_SECRET=<paste-from-above>
   PLATFORM_ROOT_SECRET=<paste-from-above>
   ```

   <Aside type="tip" title="Back up PLATFORM_ROOT_SECRET">
     Losing it makes every encrypted row unreadable — store it in a password manager.
   </Aside>

3. **Launch with Docker Compose.**

   ```bash
   docker compose up -d
   ```

   This starts the API, Console UI, and database. Wait a few seconds for migrations, then verify:

   ```bash
   curl localhost:8787/health
   ```

   Expected:
   ```json
   {"status":"ok","backends":{"db":"sqlite"},"version":"0.1.0"}
   ```

4. **Create your first agent.**

   ```bash
   export OMA_BASE="http://localhost:8787"
   export OMA_KEY="dev-test-key"

   AGENT_ID=$(curl -s -X POST "$OMA_BASE/v1/agents" \
     -H "x-api-key: $OMA_KEY" \
     -H "content-type: application/json" \
     -d '{
       "name": "hello-agent",
       "model": "claude-sonnet-4-6",
       "system": "You are a helpful assistant. Keep replies short.",
       "tools": [{"type": "agent_toolset_20260401", "default_config": {"enabled": true}}]
     }' | python3 -c "import sys,json;print(json.load(sys.stdin)['id'])")
   echo "Agent: $AGENT_ID"
   ```

5. **Send a message.**

   ```bash
   SESSION_ID=$(curl -s -X POST "$OMA_BASE/v1/sessions" \
     -H "x-api-key: $OMA_KEY" \
     -H "content-type: application/json" \
     -d "{\"agent\": \"$AGENT_ID\"}" | python3 -c "import sys,json;print(json.load(sys.stdin)['id'])")

   curl -N -X POST "$OMA_BASE/v1/sessions/$SESSION_ID/messages" \
     -H "x-api-key: $OMA_KEY" \
     -H "content-type: application/json" \
     -d '{"content": "Say hello and tell me what tools you have available."}'
   ```

   You'll see the model's reply stream token-by-token.

6. **Open the Console.**

   Open [localhost:8787](http://localhost:8787) in your browser. From here you can:

   - **Add a Model Card** — go to **Model Cards → Add** and enter your API key
   - **Create an agent** — **Agents → + New agent** with the visual builder
   - **View sessions** — **Sessions** to see all runs and monitor activity

</Steps>

<TabItem label="☁️ Cloudflare (production)">

Deploy to Cloudflare Workers for a production setup.

### Prerequisites

| Tool | Version | Why |
|---|---|---|
| [Node.js](https://nodejs.org) | 22+ | CLI and dev tooling |
| [pnpm](https://pnpm.io/installation) | 10+ | Package manager (`npm i -g pnpm`) |
| Cloudflare account | Workers Paid ($5/mo) | Durable Objects + R2 required |
| wrangler CLI | Latest | `npx wrangler login` |
| Anthropic API key | Any | Or OpenAI compatible provider |

### Steps

<Steps>

1. **Clone, install, and login.**

   ```bash
   git clone https://github.com/duyet/oma.git
   cd oma
   pnpm install
   npx wrangler login
   ```

2. **Run the setup wizard.**

   ```bash
   ./scripts/setup-cf.sh
   ```

   The wizard is fully interactive and idempotent:
   - Creates D1 databases, KV namespace, R2 buckets
   - Sets secrets (auto-generates auth secrets, prompts for `ANTHROPIC_API_KEY`)
   - Applies migrations
   - Deploys main + agent + integrations workers
   - Prints your Console URL

3. **Open the Console.**

   Visit the URL printed by the wizard (e.g. `https://oma-main.xxx.workers.dev`). Sign up — your first user becomes the tenant owner.

</Steps>

<TabItem label="🚢 Kubernetes (scale)">

Run sandbox pods on your Kubernetes cluster.

### Prerequisites

- Kubernetes v1.25+ with Helm 3
- cert-manager installed
- OMA already deployed (Cloudflare or Docker)

### Install k8s-bridge

```bash
helm repo add oma https://charts.oma.duyet.net
helm repo update

export K8S_BRIDGE_TOKEN=$(openssl rand -base64 32)

helm install oma-k8s-bridge oma/oma-k8s-bridge \
  --namespace oma-sandbox \
  --create-namespace \
  --set secret.token=$K8S_BRIDGE_TOKEN \
  --set ingress.enabled=true \
  --set ingress.host=k8s-bridge.example.com
```

### Verify

```bash
kubectl -n oma-sandbox rollout status deployment/oma-k8s-bridge
curl -H "Authorization: Bearer $K8S_BRIDGE_TOKEN" \
  https://k8s-bridge.example.com/api/v1/health
```

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| `docker compose up` fails | Port 8787 in use | Stop the conflicting process or change the port in `docker-compose.yml` |
| `health` returns 503 | Migrations not finished | Wait 10s and retry, or check `docker compose logs oma` |
| Agent creation fails with 401 | Wrong API key | Verify `OMA_KEY` matches `API_KEY` in `.env` |
| Session message returns no reply | No model card or `ANTHROPIC_API_KEY` | Add a Model Card in the Console, or set `ANTHROPIC_API_KEY` in `.env` and restart |
| `setup-cf.sh` fails | Wrangler not logged in | Run `npx wrangler login` first |
